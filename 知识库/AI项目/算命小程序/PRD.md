# 算命小程序 PRD

> **产品定位**：以"算命/命理"为载体的情绪价值小程序，对标性格分类类产品，用轻量互动+AI生成内容满足用户的好奇心与情感需求
>
> **目标上线**：2026年5月

---

## 一、产品概述

### 1.1 核心洞察
- **对标产品**：性格分类类小程序（数百万用户）
- **底层逻辑**：用户不是在求"准"，是在求"被理解"和"确定性安慰"
- **情绪价值 > 功能价值**：算命比性格测试更有"神秘感"和"传播性"

### 1.2 产品一句话
> 一个用 AI 生成个性化命理解读的小程序，让用户在 30 秒内获得"被命运眷顾"的情绪满足。

### 1.3 目标用户
| 用户画像 | 特征 | 使用场景 |
|---------|------|---------|
| 年轻职场人 | 22-30岁，工作焦虑，未来迷茫 | 午休/睡前刷到，好奇测试 |
| 情感困惑者 | 恋爱/婚姻决策期 | 感情不顺时寻求"天意"指引 |
| 社交分享者 | 喜欢发朋友圈/群聊 | 测完觉得准，转发给朋友 |

## 二、核心功能模块

### 2.1 MVP 功能（五月上线版）

#### 功能1：八字/星座快速测算
- **输入**：出生日期（公历/农历切换）+ 出生时间（可选）
- **输出**：AI生成的人格特质解读（500字以内）
- **亮点**：
  - 用"古籍引述"增加权威感（如《滴天髓》云...）
  - 正向为主，负面委婉包装（"先苦后甜"、"大器晚成"）

#### 功能2：每日运势
- **每日生成**：今日运势评分 + 3条指引（事业/感情/财运）
- **互动设计**：可"求签"——随机抽取一条签文
- **留存钩子**：订阅推送（每日8点"今日运势已更新"）

#### 功能3：缘分匹配
- **输入**：双方出生信息
- **输出**：合婚/星座配对分析
- **传播点**：情侣测完分享到朋友圈"我们的缘分指数95%"

#### 功能4：AI 智能问答
- **形式**：用户提问（"我今年能升职吗？""TA对我有意思吗？"）
- **输出**：AI结合命理框架给出"卦象式回答"
- **限制**：免费3次/天，超限引导分享/看广告解锁

### 2.2 第二版功能（上线后迭代）
- 手相/面相上传识别（视觉AI）
- 塔罗牌占卜（西方命理，吸引海外用户）
- 命理社区（用户分享经历，UGC内容）
- 大师在线咨询（真人付费服务）

## 三、用户体验流程

### 3.1 新用户路径
```
小程序打开 → 3秒动效开场（"天机泄露"氛围） → 选择测算类型
→ 输入出生信息 → 加载动画（"正在排盘..."，3-5秒制造期待）
→ 结果页（60%免费展示 + 40%模糊处理引导分享/付费）
→ 分享按钮突出（"测测你的朋友是什么命"）
```

### 3.2 老用户路径
```
订阅推送/朋友圈回流 → 首页 → 每日运势（快速消费）
→ 更多测算（深度使用）→ 付费解锁/分享解锁
```

## 四、商业模式

### 4.1 收入设计
| 层级 | 功能 | 价格 |
|-----|------|------|
| 免费 | 基础八字解读、每日运势摘要 | 0元 |
| 分享解锁 | 完整版报告、额外问答次数 | 分享到1个群/朋友圈 |
| 会员（月卡） | 无限问答、手相识别、专属大师 | 9.9-19.9元/月 |
| 单次付费 | 详细流年运势、合婚报告 | 6.9-29.9元/次 |

### 4.2 成本结构
- **AI调用**：**DeepSeek API（主力）** + Kimi API（补充），成本可控
- **服务器**：微信小程序云开发（初期低成本）
- **推广**：朋友圈广告/小红书种草（自然传播为主）

### 4.3 API 成本对比（关键）

| 模型 | 输入价格 | 输出价格 | 性价比评价 |
|-----|---------|---------|-----------|
| **DeepSeek-V3** | **¥1 / 百万 tokens** | **¥2 / 百万 tokens** | 🏆 **极度便宜，首选** |
| **DeepSeek-R1** | ¥4 / 百万 tokens | ¥16 / 百万 tokens | 推理能力强，适合复杂分析 |
| Kimi (8K) | ¥12 / 百万 tokens | ¥60 / 百万 tokens | 中文好，但贵 **10-30倍** |
| Kimi (128K) | ¥60 / 百万 tokens | ¥60 / 百万 tokens | 长文本场景用 |

**测算示例**：一次命理解读约消耗 800-1500 tokens
- DeepSeek 成本：约 **¥0.002~0.003**（几乎可以忽略）
- Kimi 成本：约 **¥0.05~0.1**（贵 20-50 倍）

**结论**：DeepSeek 便宜到离谱，建议作为主力。Kimi 可以在需要高质量长文本时偶尔调用。

## 五、技术方案

### 5.1 技术栈建议
| 层级 | 技术选型 |
|-----|---------|
| 前端 | 微信小程序原生 / Taro |
| 后端 | 微信云开发（云函数+云数据库）|
| AI生成 | **DeepSeek API**（性价比首选）/ Kimi API（中文质量优）|
| 排盘算法 | 开源八字排盘库（如 python-bazi）或自研 |
| 支付 | 微信支付 |

### 5.2 AI Prompt 策略
- **系统Prompt**：设定角色为"精通子平八字、紫微斗数的命理师"
- **生成约束**：
  - 70%正向内容 + 30%委婉提醒
  - 引用古籍增加可信度
  - 避免具体医疗/法律建议（合规）
  - 每段结尾留钩子（"完整流年分析需解锁..."）

### 5.3 合规注意
- ⚠️ **关键**：在显著位置标注"本内容仅供娱乐参考，不构成任何决策建议"
- 避免涉及：医疗诊断、投资建议、赌博预测
- 用户协议中明确"AI生成内容，非真人命理师判断"

## 八、风险与对策

| 风险 | 对策 |
|-----|------|
| 微信审核不通过（迷信内容） | 强调"娱乐参考"定位，弱化"算命"用词，用"命理文化"、"性格分析"替代 |
| AI生成内容不稳定 | Prompt工程优化 + 结果缓存（相同八字直接复用） |
| 用户觉得"不准" | 巴纳姆效应设计（模糊但正向的通用描述）+ 让用户自我验证 |
| 竞品抄袭 | 快速迭代，建立品牌认知（"XX命理"） |
