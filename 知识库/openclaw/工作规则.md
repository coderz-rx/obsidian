## 📅 周复盘规则

### 命名格式
```
YYYY-MM-DD~YYYY-MM-DD 周复盘.md
```
例如：`2026-03-16~2026-03-22 周复盘.md`

### 存储位置
```
openclaw/每日复盘/YYYY/MM/
```
（与每日复盘同目录）

### 内容结构

| 模块 | 说明 |
|------|------|
| **本周概览** | 工作主线 + 生活主题（一句话总结） |
| **工作表现** | 完成事项清单 + 效率评价 |
| **生活与成长** | 情绪管理、社交、健康运动、输入学习、琐事推进 |
| **核心收获** | 3-4条本周最重要的认知/进步 |
| **待改进点** | 不足和未完成的方面 |
| **与上周对比** | 进步/退步/持平的维度对比表（可选） |
| **下周重点** | 明确下周要落地的事项 |

### 执行规则
- 每周日复盘（或用户指定时间）
- 读取本周每日复盘内容，提炼要点
- 生成周复盘文档，存储到对应位置

## 🧠 记忆更新规则

### 核心原则
**认知变化必须经过确认才能更新记忆**

### 执行流程
1. **观察变化** → 通过对话/行为发现对你的新认知
2. **主动提出** → 向你陈述我的认知变化
3. **等待确认** → 你确认正确或纠正偏差
4. **更新记忆** → 确认后才更新 MEMORY.md 或相关记忆文件

### 禁止事项
- ❌ 擅自更新对你的认知记忆
- ❌ 基于单次对话就下结论
- ❌ 不确认就写入长期记忆
