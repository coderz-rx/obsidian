# 技术文章写作框架拆解案例

> 原文：《国际酒店AI Coding落地实践》
> 分析日期：2026-03-19

---

## 📐 文章整体结构

### 1. 钩子引入（Hook）
**"预售订单O页改动，AI生成的代码采纳率接近100%"**
- 用一个**具体成果数据**开头，立即建立可信度
- 给读者一个"这文章值得读"的理由

### 2. 写作动机（Why）
**"为什么写这篇文档"**
- 交代背景：AI Coding工具从"补全代码"进化到"理解项目上下文"
- 抛出冲突：**理想（Vibe Coding热词）vs 现实（10年+遗留系统）**
- 明确文章价值定位：记录实践、梳理全景、供团队参考

### 3. 技术全景（What）
**"AI Coding技术全景"**
系统性地铺陈6个核心概念：
- MCP（基础设施层）
- Skills（经验封装层）
- Hook（约束层）
- SubAgent（协作层）
- OpenCode（工具层）
- Spec Coding vs Vibe Coding（方法论层）

> 💡 **技巧**：每个概念都是"一句话定义 + 工程价值 + 实际场景"，不堆砌术语

### 4. 问题定义（Problem）
**"遗留系统的特殊挑战"**
列出4个核心难点：
- 上下文密度高
- 隐性规则多
- 改动影响面大
- 构建成本高

然后抛出核心判断：**AI Coding不能是"让AI自由发挥"，必须是"工程化约束下的人机协作"**

### 5. 解决框架（Framework）
**"遗留系统AI Coding的能力三角"**
用表格形式给出三层架构：

| 层级 | 核心问题 | 复用性 |
|------|----------|--------|
| 工程化约束框架 | AI怎么工作 | 高 |
| 业务知识沉淀 | AI知道什么 | 低 |
| 个人AI协作能力 | 人怎么用AI | 无 |

### 6. 方案详述（How）
**"个人方案"** — 文章核心篇幅，分5个维度展开：

- **整体思路**：AI是pair programmer，不是自主开发者
- **分层上下文管理**：全局CLAUDE.md + 服务级CLAUDE.md
- **分级工作模式**：Simple / Standard / Spec 三档，附决策树
- **权限与安全边界**：settings.local.json的显式控制
- **编译策略**：AI自主判断编译时机

> 💡 **技巧**：每个小节都有"设计决策"说明为什么这么选，而非只讲怎么做

### 7. 横向对比（Comparison）
**"与其他方案的对比"**
- 纯Vibe Coding：优势+局限
- OpenCode：优势+局限
- OpenSpec：优势+局限
- 我的方案：定位总结

> 💡 **技巧**：不贬低其他方案，而是诚实说"适合什么场景"

### 8. 适用边界（Boundary）
**"适用边界"**
明确说适用什么、不适用什么 —— **这是建立信任的关键**

### 9. 未来计划（Roadmap）
**"后续计划"**
列出5个改进方向：
- 提示词优化
- 团队共同维护
- Hook自动化
- 测试集成
- 度量体系

> 💡 **技巧**：坦诚当前短板，展示持续迭代的态度

### 10. 总结升华（Conclusion）
**一句话核心观点**：AI Coding在遗留系统上的核心矛盾是"模型能力越强，越需要工程化约束来确保产出质量"

## 🎯 值得学习的写作技巧

| 技巧 | 体现 |
|------|------|
| **数据开头** | "采纳率接近100%"立住专业形象 |
| **冲突驱动** | 理想（Vibe Coding）vs 现实（遗留系统） |
| **分层展开** | 全景→问题→框架→方案→对比→边界，层层递进 |
| **决策透明** | 每个设计都讲"为什么这么选" |
| **诚实边界** | 不夸大，明确说适用/不适用场景 |
| **承认不足** | 最后列出改进方向，展示迭代思维 |

---

## 💭 为什么写起来费劲？

这篇文章的作者有很强的**结构意识**和**读者视角**：
1. 他知道读者可能懂AI Coding，但不懂遗留系统的特殊性 → 所以花篇幅讲挑战
2. 他知道读者会想问"和其他方案有什么区别" → 专门做对比章节
3. 他知道技术方案文章最忌"看起来完美" → 主动说适用边界和待改进点

这种**"预判读者疑问并主动回答"**的结构设计，是长期写作训练出来的。

## 📌 可复用的写作模板

```
1. 钩子（数据/成果）
2. 为什么写（背景+冲突）
3. 概念铺垫（让读者跟上）
4. 问题定义（痛点）
5. 解决框架（顶层设计）
6. 方案详述（怎么做+为什么）
7. 对比分析（和竞品/其他方案）
8. 适用边界（诚实）
9. 后续计划（迭代思维）
10. 一句话总结（金句）
```
